# Профессии: создать, проверить, отредактировать

Практический рунбук оператора. Архитектура и инварианты — в
[`../docs/CONTENT_FACTORY.md`](../docs/CONTENT_FACTORY.md); здесь — только шаги и команды.

**Три правила, на которых всё держится:**
- Всё новое попадает в БД **черновиком** (`draft`). Публикует — человек, осознанно.
- **Правки человека никогда не перетираются.** Любая правка в админке/одобренная
  правка «замораживает» строку — пересев её не трогает. БД — источник истины.
- **ИИ помогает на этапе авторинга**, в работающем приложении ИИ нет.

---

## 1. Создать новую профессию (seeds — путь разработчика)

1. Сгенерируй контент промптом [`AUTHOR_PROFESSION.md`](AUTHOR_PROFESSION.md) в новом
   чате Claude Code. Он кладёт файлы в `db/seeds/curriculum/<slug>/` нужного формата
   (`status: draft`). Структура:
   ```
   db/seeds/curriculum/<slug>/
     path.yml                       # title, description, status: draft
     01-<курс>/
       course.yml                   # title, slug, status: draft
       01-<раздел>/
         section.yml                # title → станет stage урока
         <урок>.md                  # frontmatter + ЗАЧЕМ + тело + ## Задание
   ```
2. Импортируй **одной командой** (черновик, отчёт `+N new / K frozen`):
   ```bash
   bin/rails "content:import[<slug>]"     # напр. content:import[svarshchik]
   ```
   (в bash скобки — в кавычки). Без slug — импортирует всё дерево (как `db:seed`).

## 2. Проверить

```bash
bin/rails test            # машинерия (перед любой правкой кода) — должно быть зелено
bin/rails content:check   # QA-проход контента: audit + links
```
- `content:audit` — теория без блока самопроверки `> [!ПРОВЕРЬ]`.
- `content:links` — мёртвые ссылки в ресурсах (бьёт в сеть, медленно).
- **Глазами:** открой профессию в админке как черновик.
- Глубокое **ревью контента** — [`QA_REVIEW.md`](QA_REVIEW.md); ревью
  **кода/архитектуры** — [`ARCHITECTURE_REVIEW.md`](ARCHITECTURE_REVIEW.md).

## 3. Опубликовать (осознанно)

- **Через seeds:** поменяй `status: draft → published` в `path.yml`/`course.yml` →
  `bin/rails "content:import[<slug>]"` ещё раз (строка не заморожена — статус обновится).
- **Через админку:** опубликуй кнопкой на `/admin/paths` / `/admin/courses`.

---

## 4. Редактировать — три пути

| Кто | Как | Что происходит |
|-----|-----|----------------|
| **Разработчик** | правишь `.md`/`.yml` → `content:import[<slug>]` | обновляются **только** строки, которые не трогал человек; замороженные пропускаются |
| **Эксперт без доступа к репо** | `/admin/imports/new` → вставить YAML → предпросмотр → подтвердить | ложится **черновиком** (`origin: ai`), та же защита заморозки |
| **Эксперт/редактор** | `/admin/lessons` → урок → редактировать | **первая правка человека замораживает урок** (`origin: human`) — пересев его больше не трогает; каждая правка пишет **неизменяемую ревизию** (история); откат = новая ревизия |

Плюс **предложения от читателей:** «предложить правку» на уроке → редактор смотрит
на `/admin/lesson_suggestions` → одобряет → правка применяется + ревизия.

> Кто что может: `member` → `editor` (Эксперт: контент своих профессий) →
> `administrator` (всё + `/admin/users`, публикация). Подробнее — `../CLAUDE.md`.

## Шпаргалка команд

```bash
bin/rails "content:import[svarshchik]"   # импорт одной профессии (черновик)
bin/rails content:import                 # импорт всего дерева сидов
bin/rails content:check                  # QA: audit + links разом
bin/rails content:audit                  # теория без самопроверки
bin/rails content:links                  # мёртвые ссылки
bin/rails test                           # все тесты
bin/rubocop                              # линт
```
