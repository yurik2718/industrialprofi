# tools/ — кит фабрики контента

Промпты и проверки для создания качественных материалов: ИИ-черновик → проверка
экспертом → правки читателей. Всё запускается на этапе авторинга/обслуживания
(консоль Claude Code или внешний LLM); само приложение остаётся **без LLM** — ИИ
не публикует и не перетирает правки человека.

> **Начни отсюда:** [`CONTENT_FACTORY.md`](CONTENT_FACTORY.md) — канонический
> документ: архитектура фабрики (схемы, инвариант заморозки, защита slug) **и**
> пошаговый канон «как создать качественную профессию». Остальные файлы здесь —
> это инструменты, которые он связывает.

## Промпты (русские — генерируют русский контент)

- [`AUTHOR_PROFESSION.md`](AUTHOR_PROFESSION.md) — спроектировать roadmap и написать
  уроки целой профессии. **Здесь канон формата и качества урока.**
- [`DEEPEN_LESSON.md`](DEEPEN_LESSON.md) — точечно углубить один готовый урок.
- [`LESSON_IMAGES.md`](LESSON_IMAGES.md) — инфографика к урокам (Nano Banana,
  фиксированный стиль).

## Ревью (на чтение — ИИ предлагает, человек решает)

- [`QA_REVIEW.md`](QA_REVIEW.md) — судящая половина QA: читаем БД, оцениваем
  ясность / точность / глубину → **предложения эксперту**. (Механическая половина —
  `bin/rails content:check`.)
- [`ARCHITECTURE_REVIEW.md`](ARCHITECTURE_REVIEW.md) — ревью кода и архитектуры
  самой фабрики (импортёры, заморозка, модель данных) на надёжность и удобство.
