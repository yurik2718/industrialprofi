# tools/ — кит фабрики контента

Промпты и проверки для создания качественных материалов: ИИ-черновик → проверка
экспертом → правки читателей. Архитектура и инвариант — в
[`docs/CONTENT_FACTORY.md`](../docs/CONTENT_FACTORY.md).

> **Начни отсюда:** [`WORKFLOW.md`](WORKFLOW.md) — рунбук «создать / проверить /
> отредактировать профессию» (seeds, YAML, веб-редактор) с командами-шпаргалкой.

> Всё это запускается на этапе авторинга/обслуживания (Claude Code или внешний
> LLM). Само приложение остаётся **без LLM**: ИИ никогда не публикует — публикует
> эксперт; и ИИ не перетирает правки человека.

## Промпты (русские — генерируют русский контент)

- [`AUTHOR_PROFESSION.md`](AUTHOR_PROFESSION.md) — спроектировать roadmap и
  написать уроки целой профессии (структура → согласование → уроки + отчёт на
  ручную проверку ссылок и мест под картинки).
- [`DEEPEN_LESSON.md`](DEEPEN_LESSON.md) — точечно углубить один готовый урок.
- [`LESSON_IMAGES.md`](LESSON_IMAGES.md) — инфографика к урокам (Nano Banana,
  фиксированный стиль).

## Проверка качества (QA)

- **Механическая (правила):**
  - `bin/rails content:audit` — теория без блока самопроверки `> [!ПРОВЕРЬ]`.
  - `bin/rails content:links` — ссылки-ресурсы, которые перестали открываться.
- **Суждение (Claude Code):** [`QA_REVIEW.md`](QA_REVIEW.md) — читаем БД,
  оцениваем ясность / точность / глубину / уместность → **предложения эксперту**.
  В живой контент напрямую не пишем.

## Ревью кода и архитектуры

- **Архитектурное ревью (Claude Code):** [`ARCHITECTURE_REVIEW.md`](ARCHITECTURE_REVIEW.md)
  — старший архитектор с вкусом DHH проверяет код и архитектуру самой фабрики
  (импортёры, заморозка, модель данных, админ-импорт) на надёжность и удобство.
  На чтение: ревью, а не правки.

## Принцип (во всех промптах)

Польза и ясность важнее полноты — **ничего ради галочки**. Теория завершается
вопросами `> [!ПРОВЕРЬ]`; практика и схемы — только где реально помогают; ссылки
типо-уместные (нормативы/документы — для зарегулированного, протоколов, языков;
иначе лучший YouTube / Habr / стандартный инструмент).

## Как контент попадает в БД (безопасно)

Готовое попадает в БД через create-or-refresh (см. `CONTENT_FACTORY.md`):
нетронутые ИИ-черновики обновляются, правки человека — заморожены. Два пути:
- **сид-дерево `.md`** (`db/seeds/curriculum/…`) — то, что генерирует
  `AUTHOR_PROFESSION.md`;
- **одиночный YAML** → `/admin/imports/new` (предпросмотр → подтверждение).
