Практика: собираем мнемосхему насосной станции с тегами, алармами и трендом. Финал верхнего уровня — процесс обретает лицо для оператора.
---
Это практическое задание. Делается в учебной/демо-версии любой SCADA (подойдёт любой пакет — принципы одинаковы) либо подробно проектируется на бумаге, если установить SCADA нет возможности. Не привязывайся к конкретному вендору — думай концепциями: теги, динамизация, алармы, тренды.

## Задание

### Исходные данные

Используй наработки прошлых уроков: программу насосной из раздела «Программирование ПЛК», список тегов и эскиз мнемосхемы из этого раздела.

## Шаг 1. Теги

Создай (или выпиши) теги: `Давление`, `Уровень_бак`, `Насос_состояние`, `Режим_авто`, `Авария_сухой_ход`. Для каждого укажи тип (аналоговый/дискретный) и источник на стороне ПЛК.

## Шаг 2. Мнемосхема

Собери экран:
1. приёмный бак с динамической заливкой по тегу `Уровень_бак`;
2. насос, меняющий цвет по `Насос_состояние` (серый/зелёный);
3. напорный бак и труба по ходу потока;
4. цифровой индикатор давления (число + единицы «бар»);
5. индикатор режима «АВТО» и индикатор аварии «сухой ход».

Примени правило цвета: норма — спокойно, авария — заметно.

## Шаг 3. Алармы

Настрой (или опиши) тревоги:
- `Авария_сухой_ход` — приоритет «авария», с квитированием;
- `Давление` выше порога — приоритет «предупреждение»;
- потеря связи с ПЛК — «авария».

## Шаг 4. Тренд

Добавь тренд с двумя кривыми: `Давление` и `Уровень_бак`. Настрой период отображения (например, последний час) — чтобы оператор видел динамику.

## Шаг 5. Управление

Добавь на экран элементы управления: кнопки ПУСК/СТОП режима и (если SCADA позволяет) поле уставки уровня. Помни: команда с мнемосхемы — это запись в тег, который связан с регистром/входом ПЛК.

## Критерии самопроверки

Работа зачтена, если:
- каждый элемент мнемосхемы привязан к конкретному тегу, и ты можешь проследить его путь до физического сигнала ПЛК;
- в штатном режиме экран спокойный, авария бросается в глаза;
- тревога «сухой ход» требует квитирования и попадает в журнал событий;
- тренд показывает историю давления и уровня;
- человек, не знающий процесс, за несколько секунд понимает по экрану: работает ли насос, сколько воды, есть ли авария.

> [!СОВЕТ] Поздравляю — ты только что прошёл весь путь сигнала своими руками: датчик → вход ПЛК → алгоритм → тег → мнемосхема оператора. Это и есть работа инженера АСУ ТП в миниатюре. Сохрани проект целиком (программа ПЛК + мнемосхема) — это первый экспонат твоего портфолио.
