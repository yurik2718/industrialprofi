Маршрутизация — механизм, позволяющий ПЛК и SCADA-серверу работать в разных подсетях и всё равно обмениваться данными через L3-коммутатор или маршрутизатор.
---
## Ментальная модель: GPS в городе

Коммутатор (L2) — это почтальон внутри одного дома: он знает все квартиры (MAC-адреса), но за пределы дома не выходит. Маршрутизатор — это водитель с GPS: он знает маршруты между районами (подсетями) и прокладывает путь от одного здания к другому.

Каждый раз, когда пакет нужно доставить в другую подсеть, он приходит к маршрутизатору. Маршрутизатор смотрит в **таблицу маршрутизации** и решает: «в подсеть 10.10.30.0/26 — через интерфейс Gi0/1».

## Таблица маршрутизации

Таблица маршрутизации — главный инструмент маршрутизатора. Каждая запись содержит:
- **Сеть назначения** с маской
- **Следующий узел** (next-hop) или исходящий интерфейс
- **Метрика** — «стоимость» пути (чем меньше, тем предпочтительнее)
- **Источник** — откуда маршрут получен: `C` (Connected), `S` (Static), `O` (OSPF)

```
Сеть назначения   Маска     Следующий узел    Интерфейс   Метрика   Источник
10.10.10.0       /24       —                 Gi0/0       0         C (Connected)
10.10.20.0       /26       —                 Gi0/1       0         C (Connected)
10.10.30.0       /26       10.10.20.1        Gi0/1       1         S (Static)
0.0.0.0          /0        192.168.1.1       Gi0/2       1         S (Default)
```

## Статическая маршрутизация

Самый простой способ: администратор вручную вписывает маршруты. На промышленных объектах — типичный выбор для сетей АСУ ТП: мало подсетей, мало изменений, полный контроль.

```
ip route 10.10.30.0 255.255.255.192 10.10.20.1
```

Эта команда говорит маршрутизатору: «для трафика в сеть 10.10.30.0/26 — отправляй пакеты на следующий узел 10.10.20.1».

> [!ПРИМЕР]
> SCADA-сервер (IP: 10.10.20.5) хочет прочитать данные из ПЛК (IP: 10.10.30.10).
>
> 1. SCADA видит: пункт назначения 10.10.30.10 — не в её подсети (10.10.20.0/26). Отправляет пакет на **шлюз по умолчанию** 10.10.20.254.
> 2. L3-коммутатор (шлюз) получает пакет, смотрит таблицу маршрутизации: 10.10.30.0/26 → интерфейс VLAN 30.
> 3. Коммутатор пересылает пакет в VLAN 30. ПЛК получает запрос и отвечает.
> 4. Ответ идёт обратно: ПЛК → шлюз → SCADA (маршрут обратный по той же логике).

## Маршрут по умолчанию (default route)

Маршрут `0.0.0.0/0` — «если не знаешь куда — туда». Самый широкий маршрут, самый низкий приоритет. На производстве это обычно выход в корпоративную сеть или интернет.

Для промышленной сети АСУ ТП default route **часто намеренно не настраивают** или ограничивают межсетевым экраном — чтобы ПЛК не мог «случайно» выйти в интернет.

## Динамическая маршрутизация: OSPF

Когда сеть большая (десятки подсетей, несколько маршрутизаторов), статические маршруты становятся трудноуправляемыми. **OSPF** (Open Shortest Path First, RFC 2328) — протокол динамической маршрутизации, где маршрутизаторы автоматически обмениваются информацией о топологии и строят оптимальный маршрут.

На промышленных объектах OSPF обычно используют только в **IT-части** (корпоративная сеть). В **OT-части** — почти всегда статические маршруты: меньше сложности, предсказуемое поведение.

> [!ВАЖНО]
> В промышленной сети **не запускай OSPF или другие протоколы динамической маршрутизации в сегментах ПЛК**. Служебный трафик этих протоколов (hello-пакеты каждые 10 секунд) создаёт дополнительную нагрузку на ПЛК, которые обрабатывают всё широковещание L3. Кроме того, динамическая маршрутизация означает непредсказуемое изменение путей — это неприемлемо для детерминированных промышленных протоколов.

## L3-коммутатор vs маршрутизатор

На производстве роль маршрутизатора между VLAN чаще всего берёт **L3-коммутатор** (коммутатор с функцией маршрутизации). Он быстрее обычного маршрутизатора для внутренней маршрутизации и стоит дешевле.

| | L3-коммутатор | Маршрутизатор |
|---|---|---|
| **Маршрутизация VLAN** | Да (аппаратная) | Да |
| **WAN-интерфейсы** | Нет | Да (PPP, ADSL, LTE) |
| **Скорость L3-форвардинга** | Очень высокая | Ниже (программная) |
| **Межсетевой экран** | Базовый ACL | Как правило, нет (нужен отдельный) |
| **Типичное применение в OT** | Между VLAN ПЛК/SCADA | На границе OT/IT или к WAN |
