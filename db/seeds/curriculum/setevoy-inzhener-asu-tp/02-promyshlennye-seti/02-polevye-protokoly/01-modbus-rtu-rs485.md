## Ментальная модель: диспетчер и рабочие

Modbus работает по принципу **мастер/слейв** (master/slave). Есть один диспетчер (мастер — обычно ПЛК или SCADA) и несколько рабочих (слейвы — датчики, приводы, контроллеры). Диспетчер **всегда задаёт вопросы**, рабочие **только отвечают**. Сами инициативу рабочие не проявляют никогда.

По одной физической шине RS-485 — до **247 слейвов**, каждый со своим адресом от 1 до 247. Адрес 0 — широковещательный: мастер отправляет команду всем сразу, но ответа не ожидает.

## RS-485: физика шины

**RS-485** — дифференциальный интерфейс передачи данных. Вместо одного провода он использует два: A (D+) и B (D−). Передатчик создаёт разность напряжений между ними: логическая «1» — A положительнее B, логический «0» — наоборот.

Дифференциальная схема **устойчива к электромагнитным помехам** — именно поэтому RS-485 выжил в промышленности на десятилетия.

| Параметр | Значение |
|----------|---------|
| Максимальная длина шины | 1200 м (при скорости 100 кбит/с) |
| Максимальная скорость | 10 Мбит/с (на коротком расстоянии) |
| Типовые скорости | 9600, 19200, 38400, 57600, 115200 бит/с |
| Максимум устройств на шине | 32 нагрузочных блока (стандарт), 247 устройств Modbus |
| Топология | Шина (линия) с терминаторами на концах |
| Режим | Полудуплекс (половинный дуплекс) |

> [!ВАЖНО]
> На концах шины RS-485 **обязательно устанавливай терминирующие резисторы** — 120 Ом между линиями A и B. Без них сигнал отражается от конца кабеля и создаёт помехи. Если видишь Modbus-ошибки CRC — первым делом проверяй терминаторы.

## Структура кадра Modbus RTU

Кадр RTU (Remote Terminal Unit) — компактный бинарный формат. Байты идут один за другим без разделителей; начало и конец кадра определяются паузой на шине ≥3,5 символа.

```
[Адрес слейва: 1 байт | Код функции: 1 байт | Данные: N байт | CRC: 2 байта]
```

**Основные коды функций:**

| Код | Функция | Что читает/пишет |
|-----|---------|-----------------|
| 01 | Read Coils | Дискретные выходы (биты): катушки реле |
| 02 | Read Discrete Inputs | Дискретные входы (биты): датчики «вкл/выкл» |
| 03 | Read Holding Registers | 16-битные регистры хранения (аналоговые данные) |
| 04 | Read Input Registers | 16-битные входные регистры (только чтение) |
| 05 | Write Single Coil | Записать один бит |
| 06 | Write Single Register | Записать один регистр |
| 16 | Write Multiple Registers | Записать несколько регистров |

## Адресация регистров: «пятизначные адреса»

В документации производителей часто используется «пятизначная» нотация:
- **0xxxx** — Coils (FC01/05): дискретные выходы
- **1xxxx** — Discrete Inputs (FC02): дискретные входы
- **3xxxx** — Input Registers (FC04): аналоговые входы, только чтение
- **4xxxx** — Holding Registers (FC03/16): аналоговые данные, чтение и запись

> [!ПРИМЕР]
> Частотный привод (ЧРП) подключён к ПЛК по Modbus RTU, адрес слейва 5.
>
> В документации привода написано: «регистр 40001 — уставка частоты, 40002 — текущая частота».
>
> ПЛК отправляет запрос:
> - Адрес: 05 (слейв 5)
> - Код функции: 03 (Read Holding Registers)
> - Начальный адрес: 0x0000 (регистр 1 в диапазоне 4xxxx = «40001»)
> - Количество регистров: 02 (читаем 40001 и 40002)
> - CRC: рассчитывается по всему пакету
>
> Привод отвечает двумя 16-битными значениями. Если уставка частоты 50 Гц, а масштаб = 0.1 Гц/LSB — значение в регистре: 500 (0x01F4).

## Типичные ошибки и диагностика

| Симптом | Вероятная причина |
|---------|-----------------|
| CRC error | Нет терминатора, помехи, неправильная скорость |
| No response | Неверный адрес слейва, устройство выключено, обрыв A или B |
| Exception response (код функции + 0x80) | Неверный код функции, адрес регистра за диапазоном, устройство не готово |
| Intermittent errors | Слишком длинная шина без повторителя, «паразитная» ёмкость кабеля |

> [!СОВЕТ]
> Для диагностики Modbus RTU используй **qModMaster** (бесплатный, Windows/Linux) или **Modbus Poll**. Они позволяют отправлять произвольные запросы к устройству и видеть сырые байты ответа — незаменимо при первичной наладке и поиске неисправностей. Подключись через USB/RS-485-конвертер напрямую к шине.
