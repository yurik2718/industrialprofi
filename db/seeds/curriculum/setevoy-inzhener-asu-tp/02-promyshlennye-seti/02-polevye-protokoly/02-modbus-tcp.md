Modbus TCP — это тот же Modbus, завёрнутый в TCP/IP: тот же принцип регистров и функций, но поверх Ethernet вместо RS-485, порт 502, доступен из любой точки сети.

## От RS-485 к Ethernet: что изменилось

Modbus TCP появился в 1999 году как «Modbus RTU поверх TCP/IP». Идея проста: взять привычный протокол Modbus и заменить физику RS-485 + кадровую структуру RTU на Ethernet + TCP.

Что **осталось прежним**:
- Регистровая модель: те же 0xxxx, 1xxxx, 3xxxx, 4xxxx
- Коды функций (01–16 и другие)
- Принцип мастер/клиент → слейв/сервер: клиент всегда инициирует запрос

Что **изменилось**:
- Вместо RS-485 — обычный Ethernet-кабель и IP-адреса
- Нет ограничения на 247 устройств — количество определяется сетью
- Нет адреса устройства в пакете (его заменяет IP-адрес); поле «Unit Identifier» сохранено для совместимости с шлюзами RTU→TCP
- Вместо CRC — надёжность обеспечивает TCP
- Стандартный порт: **502**

## MBAP-заголовок

Modbus TCP добавляет к PDU (Protocol Data Unit) специальный 6-байтный заголовок — **MBAP** (Modbus Application Protocol):

```
[Transaction ID: 2 | Protocol ID: 2 | Length: 2 | Unit ID: 1 | FC: 1 | Data: N]
      ↑                   ↑              ↑             ↑
  Идентификатор     Всегда 0x0000    Байт далее    Адрес слейва
  транзакции        (Modbus = 0)     (Unit+FC+Data) (для шлюзов)
```

> [!ПРИМЕР]
> Клиент (SCADA) читает 2 holding-регистра из ПЛК (IP 10.10.30.10) начиная с адреса 0x0000:
>
> TCP-соединение: источник → 10.10.30.10:502
>
> Запрос (hex):
> ```
> 00 01   — Transaction ID = 1
> 00 00   — Protocol ID = 0 (Modbus)
> 00 06   — Length = 6 байт далее
> 01      — Unit ID = 1
> 03      — Function Code 03 (Read Holding Registers)
> 00 00   — Начальный адрес = 0
> 00 02   — Количество регистров = 2
> ```
>
> Ответ ПЛК (если значения 500 и 0):
> ```
> 00 01 00 00 00 07 01 03 04 01 F4 00 00
>                              ↑  ↑  ↑  ↑  ↑
>                           Length=7 FC=03 ByteCount=4  Reg1=500(0x01F4) Reg2=0
> ```

## Безопасность Modbus TCP

Modbus TCP **не имеет встроенной аутентификации и шифрования**. Любой, кто имеет доступ к сети на порт 502, может читать и **записывать** данные в ПЛК. Это означает:

- Запись произвольных значений в управляющие регистры → изменение уставок, остановка приводов
- Чтение технологических параметров → промышленный шпионаж
- Отправка некорректных пакетов → возможный отказ устройства

> [!ОПАСНО]
> Никогда не открывай порт 502 (Modbus TCP) на межсетевом экране между OT-сетью и корпоративной или интернет-сетью. Это прямой путь к несанкционированному управлению технологическим процессом. Если нужен удалённый доступ — только через защищённый VPN-канал с аутентификацией.

## Практика: работа с Modbus Poll

Modbus Poll — де-факто стандарт для диагностики и отладки Modbus TCP/RTU на Windows. Позволяет:
- Подключаться к любому Modbus TCP-серверу по IP:502
- Читать и записывать регистры любых типов
- Видеть сырые байты запросов и ответов
- Сохранять конфигурацию опроса

## Задание

**Цель**: подключиться к Modbus TCP-устройству (реальному или симулятору) и прочитать данные.

**Понадобится:**
- ПК с Windows или Linux
- Modbus Poll (бесплатный trial на 30 дней) или qModMaster (бесплатный)
- Modbus TCP-симулятор: **ModRSsim2** (бесплатный, Windows) или онлайн-симулятор diagslave (Modbusdriver, бесплатный CLI)

**Шаги:**
1. Запусти ModRSsim2 — он сразу слушает порт 502 на localhost (127.0.0.1).
2. Открой Modbus Poll → Connection → Connect → TCP/IP → Host: 127.0.0.1, Port: 502.
3. Нажми Setup → Read/Write Definition → FC03, Address: 0, Quantity: 10 → OK.
4. В окне симулятора установи значение в регистр 40001 = 1234 (двойной клик на ячейке).
5. Убедись, что Modbus Poll показывает значение 1234 в регистре 0.
6. Попробуй записать значение: Setup → Read/Write Definition → FC16, Address: 0, Quantity: 1 → OK → щёлкни правой кнопкой по регистру → «Set Preset Register».

**Что сдать**: запиши в журнал:
- Что увидел в окне «Communication Traffic» (сырые байты запросов)
- Как выглядит MBAP-заголовок прочитанного пакета (можешь перевести hex в десятичные значения полей)
- Что произошло при записи — изменилось ли значение в симуляторе?

Если нет Windows — выполни задание письменно: расшифруй следующий пакет Modbus TCP по байтам:
`00 05 00 00 00 06 01 03 00 0A 00 05`
