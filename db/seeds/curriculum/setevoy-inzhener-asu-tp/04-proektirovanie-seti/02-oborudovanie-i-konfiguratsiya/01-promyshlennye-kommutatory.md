## Чем промышленный коммутатор отличается от офисного

На первый взгляд коммутатор — это коммутатор. Ethernet-порты, MAC-таблица, VLAN. Но в промышленном применении разница принципиальная:

| Характеристика | Офисный | Промышленный |
|----------------|---------|-------------|
| **Диапазон температур** | 0…+40°C | −40…+70°C (extended) |
| **Вибростойкость** | Нет требований | IEC 60068-2-6 (синусоидальная), IEC 60068-2-27 (удар) |
| **ЭМ-совместимость** | Базовая | EN 55032 Class A, IEC 61000-4-x (промышленный уровень) |
| **Монтаж** | Стойка 19" или настольный | DIN-рейка (35 мм), 19" или стеновой |
| **Питание** | AC 220 В | DC 24 В (типично), дублированное питание |
| **Диагностика** | Минимальная | Аварийные реле, сигнальные выходы, SNMP/PROFINET |
| **Поддержка RT-протоколов** | Нет | PROFINET RT/IRT, EtherNet/IP, MRP, PTP |
| **MTBF** | 5–7 лет | 15–20 лет (Siemens SCALANCE > 200 000 часов) |
| **Срок поставки запчастей** | 3–5 лет | 10+ лет (Long-term availability) |

## Основные производители и серии

### Siemens SCALANCE X

Наиболее распространённая серия на российских производствах с оборудованием Siemens. Три основных линейки:

| Серия | Назначение | Порты |
|-------|-----------|-------|
| **SCALANCE XB** | Неуправляемый, базовый | 8–16 порт, Fast/Gigabit |
| **SCALANCE XC** | Управляемый (L2), основная рабочая лошадка | 8–24 порт, SFP, PROFINET |
| **SCALANCE XF** | Компактный, для монтажа непосредственно на устройство | 4–8 порт |
| **SCALANCE XM** | Управляемый с маршрутизацией (L3), модульный | до 48 порт |

Конфигурирование: через Web-интерфейс, SINEMA или Command Line Interface (CLI).

### Moxa EDS-Series

Хорошее соотношение цена/качество, популярны при строительстве новых OT-сетей не завязанных на экосистему Siemens:
- **EDS-308/EDS-405A**: 5–8 портов, неуправляемые/управляемые, DIN-рейка
- **EDS-G508E/G509E**: Gigabit, управляемые, кольцо, PROFINET-агент

### Phoenix Contact FL Switch

Широкий выбор форм-факторов, хорошая поддержка промышленных протоколов. Особенность: очень простая конфигурация для нетехнических специалистов (мастер-настройка через Web).

## Что обязательно настроить перед вводом в эксплуатацию

1. **Смени пароли по умолчанию** — это первое действие. Заводские пароли известны всем.
2. **Назначь IP-адрес и hostname** из IP-плана объекта.
3. **Включи RSTP/Rapid Spanning Tree** на всех портах (кроме portfast для конечных устройств).
4. **Настрой VLAN** по схеме сегментации.
5. **Включи LLDP** — для обнаружения топологии в Proneta и TIA Portal.
6. **Настрой NTP** — синхронизация времени (критично для логов и PTP).
7. **Включи SNMPv3** с аутентификацией (не SNMPv1/v2c — они без шифрования).
8. **Отключи неиспользуемые порты** и сервисы (Telnet, HTTP — только HTTPS/SSH).
9. **Настрой port security или 802.1X** для критичных портов.
10. **Сохрани конфигурацию** и сделай резервную копию.

> [!ОПАСНО]
> Никогда не оставляй коммутатор с заводскими учётными данными (admin/admin, admin/scalance и т.п.) в работающей сети. Промышленные сети регулярно сканируются — любое устройство с дефолтными паролями будет скомпрометировано.

## Типичная конфигурация VLAN на SCALANCE (CLI-фрагмент)

```
# Создание VLAN
vlan 10
  name Profinet-Controllers
vlan 20
  name SCADA-Servers

# Назначение портов
interface gigabitethernet 0/1
  switchport mode access
  switchport access vlan 10        ! ПЛК Siemens S7-1500

interface gigabitethernet 0/5
  switchport mode access
  switchport access vlan 20        ! SCADA-сервер

interface gigabitethernet 0/8
  switchport mode trunk
  switchport trunk allowed vlan 10,20  ! Аплинк к L3-коммутатору
```

> [!СОВЕТ]
> Для быстрой диагностики используй **LLDP Neighbor Table** — она покажет, какие устройства подключены к каким портам, с MAC-адресами и именами. На SCALANCE: `show lldp neighbor`. Это первый инструмент, когда нужно понять топологию незнакомого шкафа.
