VLAN-сегментация сети АСУ ТП — главный практический навык проектирования: правильная разбивка изолирует трафик ПЛК от корпоративной сети, ограничивает воздействие при инциденте и улучшает производительность RT-протоколов.

## Принцип проектирования VLAN-сегментации для OT

Разбивка на VLAN — это не просто технический приём, а **реализация зонирования безопасности** (по IEC 62443-3-2). Ключевой принцип: устройства в одном VLAN доверяют друг другу; трафик между разными VLAN проходит через контролируемую точку (L3-коммутатор или межсетевой экран).

**Правило сегментации OT-сети:**
1. Раздели устройства по **функциональному уровню** (Purdue): уровень 0–1 отдельно от 2–3.
2. Раздели по **критичности**: управляющие ПЛК отдельно от HMI и историков.
3. Раздели **RT-трафик** (Profinet, EtherNet/IP) от обычного IP-трафика (SCADA, мониторинг).
4. Выдели **управление инфраструктурой** (коммутаторы, маршрутизаторы) в отдельный VLAN.

## Важные правила конфигурации

**VLAN 1 — зло**: никогда не используй VLAN 1 для рабочего трафика. Это дефолтный VLAN на всех коммутаторах; оставь его пустым или назначь для «карантина» непознанных устройств.

**Native VLAN на trunk-портах**: по умолчанию кадры без тега на trunk-порту попадают в native VLAN (обычно VLAN 1). Это потенциальная уязвимость (VLAN hopping). Настрой native VLAN отличным от VLAN 1 и без рабочих устройств.

**Port security**: на портах с ПЛК и HMI ограничь количество MAC-адресов (1–2 на порт) или привяжи конкретные MAC. Подключение неизвестного устройства → порт блокируется.

## Практическая конфигурация VLAN-сегментации

Для этого практического урока разберём конфигурацию трёх коммутаторов: Core L3 (SCALANCE XM408) и двух коммутаторов доступа (SCALANCE XC216).

**Core L3 коммутатор:**
```
! Создание VLAN и SVI
vlan 10
vlan 20
vlan 30
vlan 99
!
interface vlan 10
  ip address 10.10.10.1 255.255.255.192
interface vlan 20
  ip address 10.10.20.1 255.255.255.192
interface vlan 30
  ip address 10.10.30.1 255.255.255.192
interface vlan 99
  ip address 10.10.99.1 255.255.255.240
!
! Аплинк к межсетевому экрану
interface gi0/1
  switchport mode trunk
  switchport trunk allowed vlan 10,20,30,99
```

**Коммутатор доступа (шкаф ПЛК):**
```
! Порты для ПЛК (VLAN 10, Profinet RT)
interface gi0/1
  switchport mode access
  switchport access vlan 10
  spanning-tree portfast        ! Конечные устройства — мгновенный переход в forwarding
  storm-control broadcast level 20  ! Ограничение широковещания

! Аплинк к Core
interface gi0/8
  switchport mode trunk
  switchport trunk allowed vlan 10
  no spanning-tree portfast     ! Trunk не может быть portfast
```

## Задание

**Цель:** спроектировать и задокументировать VLAN-схему для реального объекта.

**Понадобится:**
- Лист бумаги или draw.io (бесплатно, онлайн: diagrams.net)
- IP-план из предыдущих уроков

**Шаги:**
1. Возьми объект из задания урока «Модель Purdue» (или создай новый).
2. Распредели все устройства по VLAN согласно принципам из урока.
3. Для каждого VLAN: присвой номер, название, подсеть (используй IP-план из урока адресации).
4. Нарисуй логическую схему: коммутаторы, VLAN-полосы, какие порты в каком VLAN. Покажи, где trunk-порты.
5. Запиши конфигурацию для одного коммутатора доступа: создание VLAN, назначение портов, настройка trunk.

**Что сдать:** таблица VLAN (номер/назначение/подсеть/устройства) + текстовое описание принятых решений. Нарисованную схему опиши словами в журнале (какие коммутаторы, как они соединены, какие VLAN на каком коммутаторе).
