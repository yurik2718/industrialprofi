DMZ между IT и OT — это не опциональное «улучшение безопасности», а единственная архитектурная схема, позволяющая передавать данные в корпоративную сеть без прямого доступа к ПЛК: без неё любой взломанный офисный компьютер получает путь к промышленному оборудованию.
---
## Почему прямое соединение IT↔OT опасно

В классической схеме без DMZ корпоративная сеть (IT) подключена к OT-сети напрямую через маршрутизатор или L3-коммутатор. SCADA или историк данных получает данные от ПЛК — всё работает.

Проблема: **при компрометации IT-системы** атакующий получает прямой сетевой доступ к OT-сегменту. Нет барьера. Это то, что произошло при атаке Colonial Pipeline (2021) — через скомпрометированный VPN-аккаунт атакующие добрались до OT-сети.

## Архитектура DMZ для OT

**DMZ (Demilitarized Zone)** — изолированный сетевой сегмент между двумя зонами с разными уровнями доверия. Он действует как «шлюз данных» с явными правилами обмена.

```
Корпоративная сеть (IT)
         │
    [Firewall IT]
         │
       [DMZ]
      ┌──────────┐
      │ Historian│  ← зеркало данных от ПЛК (только чтение)
      │ Gateway  │  ← OPC UA Proxy / Data Diode
      │ Jump     │  ← защищённая станция для удалённого доступа
      │  Server  │
      └──────────┘
         │
    [Firewall OT]
         │
  OT-сеть (ПЛК, ET 200SP, HMI)
```

**Ключевое правило DMZ для OT:**
- Трафик из OT → DMZ: разрешён (данные идут вверх)
- Трафик из IT → DMZ: разрешён (только на конкретные серверы в DMZ)
- Трафик из IT → OT напрямую: **запрещён полностью**
- Трафик из OT → IT напрямую: **запрещён полностью**

Данные «перекладываются» в DMZ как промежуточный буфер, но прямого маршрута между IT и OT нет.

## Что размещается в DMZ

| Компонент | Назначение |
|-----------|-----------|
| **Historian-зеркало** | Принимает данные от ПЛК (Modbus, OPC UA) и делает их доступными для корпоративного historian без прямого доступа к OT |
| **OPC UA Gateway (Proxy)** | Транслирует OPC UA-запросы из IT в OT с фильтрацией: только чтение, только разрешённые узлы |
| **Jump Server / Bastion** | Единственная точка доступа для операторов IT и подрядчиков в OT; всё пишется в лог |
| **Antivirus / WSUS сервер** | Обновления ПО для OT-систем проходят через DMZ, не напрямую из интернета |
| **File Exchange Server** | Безопасная передача файлов (проекты ПЛК, обновления, архивы) с проверкой |

## Межсетевые экраны для OT

Стандартный IT-firewall не подходит для OT по двум причинам:
1. Не понимает промышленные протоколы (Modbus, PROFINET, EtherNet/IP) — не может фильтровать по Function Code, Object ID и т.д.
2. Не рассчитан на DIN-rail монтаж, температурный диапазон промышленных объектов, 24V DC питание.

**OT-capable firewalls** — межсетевые экраны с Deep Packet Inspection (DPI) для промышленных протоколов:

| Производитель | Продукт | DPI-протоколы |
|--------------|---------|---------------|
| Siemens | SCALANCE SC-600 (SC622-2C, SC632-2C, SC636-2C) | Modbus TCP, PROFINET, OPC UA, S7 |
| Phoenix Contact | FL mGuard | Modbus TCP, EtherNet/IP, S7 |
| Fortinet | FortiGate Rugged | Modbus TCP, DNP3, IEC 104 |
| Check Point | Quantum Rugged | Modbus, PROFINET, OPC UA |
| Cisco | ISA3000 | Modbus, DNP3, EtherNet/IP |

> [!ПРИМЕР]
> Настройка политики SCALANCE SC-636 между DMZ и OT-сетью:
>
> Разрешить:
> - Из DMZ (Historian) → OT (ПЛК): Modbus TCP, только FC03/04 (только чтение), только порт 502, только с IP historian
> - Из OT (ПЛК) → DMZ (Historian): запрещено (данные тянет historian, не пушит ПЛК)
>
> Запрещено всё остальное.
>
> Результат: если historian-сервер в DMZ скомпрометирован, атакующий не сможет отправить запись (FC06/16) в ПЛК — firewall заблокирует по Function Code.

## Data Diode — аппаратная однонаправленность

Для наиболее критических объектов (категория 1 КИИ, АЭС, ХОПО) используют **Data Diode** (однонаправленный шлюз):

Это аппаратное устройство с физически однонаправленным оптическим каналом — данные могут идти **только** из OT в DMZ, обратное направление физически невозможно. Никаким ПО это не обходится — нет обратного оптического пути.

Решения: Waterfall Security Solutions, Owl Cyber Defense, OwlIntel, российские аналоги — АПКШ «Континент» ОД.

> [!ВАЖНО]
> Межсетевой экран уровня OT — это не замена VLAN-сегментации, а дополнение. Правило: сначала сегментация (VLAN, физическое разделение) → потом фильтрация на границе зон (firewall) → потом мониторинг трафика (IDS). Все три слоя нужны одновременно.
