## Почему OT-системы стали мишенью

До 2010 года господствовало убеждение: «промышленные системы безопасны, потому что изолированы». Air gap — физическая изоляция от интернета — казался надёжной защитой.

Stuxnet разрушил этот миф в 2010 году. Оказалось:
- **Air gap преодолевается**: через USB-накопитель, ноутбук подрядчика, CD с обновлением ПО
- **Протоколы OT не имеют защиты**: Modbus, PROFIBUS, IEC 104 разрабатывались для надёжности, не безопасности
- **Конвергенция IT/OT создала новые пути**: VPN для удалённого доступа, корпоративные сети, IIoT-устройства

Теперь OT-системы атакуют по четырём основным причинам: **шпионаж** (утечка технологий), **вымогательство** (остановка производства → выкуп), **диверсия** (вывод из строя), **геополитические цели** (атаки на критическую инфраструктуру).

## Три landmark-инцидента, которые изменили отрасль

### Stuxnet (2010)

**Что произошло**: вирус, разработанный предположительно США и Израилем, уничтожил 1000 центрифуг иранского завода по обогащению урана в Натанзе.

**Как работал**: распространялся через USB, использовал 4 zero-day уязвимости Windows. После заражения Windows-компьютера искал Siemens Step 7 (ПО для программирования ПЛК S7-315 и S7-417). При обнаружении изменял программу ПЛК так, чтобы центрифуги вращались с аномальными скоростями — разрушая себя. При этом SCADA показывала операторам нормальные значения — маскировка.

**Урок**: ПЛК — это компьютер с исполняемым кодом. Злоумышленник с доступом к ПО для программирования может изменить поведение производственного процесса так, что операторы об этом не узнают.

### Industroyer / CrashOverride (2016)

**Что произошло**: атака на украинскую энергосистему — отключение электричества в Киеве на 1 час (декабрь 2016). Атрибутируется группировке Sandworm.

**Как работал**: модульный вредоносный код с плагинами под конкретные OT-протоколы: IEC 101, IEC 104, IEC 61850 (MMS), OPC DA. Модуль просто посылал команды открытия выключателей на подстанциях по промышленным протоколам — без взлома устройств.

**Урок**: зная промышленные протоколы (они открытые), атакующий может напрямую управлять оборудованием. Протокол без аутентификации = открытая дверь.

### TRITON / TRISIS (2017)

**Что произошло**: атака на нефтехимическое предприятие Саудовской Аравии (предположительно SABIC). Атакующие добрались до **Safety Instrumented System (SIS)** Triconex — систем аварийной защиты.

**Как работал**: вредоносный код пытался перепрограммировать SIS, чтобы при реальной аварии система не сработала (или сработала неправильно). Первая попытка провалилась из-за ошибки — SIS перешла в failsafe и завод аварийно остановился. Это и раскрыло атаку.

**Урок**: цель атак сместилась с остановки производства к созданию условий для физической катастрофы (пожар, взрыв, выброс опасных веществ).

## Основные векторы атак на OT

| Вектор | Описание | Пример |
|--------|----------|--------|
| **Ноутбук подрядчика** | Поставщик подключает ноутбук для ПНР — на нём вирус | Stuxnet через Step 7 |
| **Удалённый доступ** | VPN/RDP без MFA → скомпрометированные учётные данные | Colonial Pipeline (2021) |
| **IT→OT lateral movement** | Взлом офисной сети → двигаются вглубь к OT | Norsk Hydro (2019) |
| **Уязвимые SCADA** | Устаревшие ОС (Windows XP/2003) без патчей | WannaCry на заводах |
| **Плохая сегментация** | ПЛК «видно» из корпоративной сети напрямую | Большинство инцидентов |
| **Дефолтные пароли** | admin/admin на коммутаторах и SCADA | Постоянная проблема |

> [!ВАЖНО]
> Особое место среди векторов занимают **ноутбуки подрядчиков и временные подключения**. На каждом объекте, где ты работаешь — убедись, что подключаемые инженерные станции проверены антивирусом и не содержат несанкционированного ПО. Это ответственность сетевого инженера, не только ИБ-отдела.

## Последствия инцидентов в промышленности

В отличие от IT-инцидентов (утечка данных, простой сайта), OT-инциденты могут вести к:
- **Физическому ущербу**: поломка оборудования, пожар, взрыв
- **Экологическим катастрофам**: разлив нефти, выброс химических веществ
- **Угрозе жизни**: отключение систем жизнеобеспечения или систем аварийной защиты
- **Крупным финансовым потерям**: Norsk Hydro оценил ущерб от атаки в 40–45 млн долларов
