IEC 62443 — международный стандарт кибербезопасности промышленных систем: его концепция зон и каналов (zones and conduits) — это практический инструмент для проектирования защищённой OT-сети, а не абстрактная теория.
---
## Структура серии IEC 62443

**IEC 62443** — серия стандартов ISA/IEC для безопасности промышленных систем автоматизации и управления (IACS). Состоит из 4 групп:

| Группа | Кому | Что описывает |
|--------|------|---------------|
| **62443-1-x** (General) | Всем участникам | Общие концепции, терминология, метрики |
| **62443-2-x** (Policies & Procedures) | Владельцу активов | Процессы управления безопасностью, программы патчинга |
| **62443-3-x** (System) | Интегратору | Оценка рисков, требования к системе, зоны и каналы |
| **62443-4-x** (Component) | Производителю | Требования к компонентам и продуктам |

Для сетевого инженера OT ключевые части: **62443-3-2** (зонирование и оценка рисков) и **62443-3-3** (технические требования безопасности).

В России адаптированы как ГОСТ Р МЭК 62443-2-1-2015, ГОСТ Р 56498-2015 (IEC/PAS 62443-3) и другие. Полное соответствие между ГОСТ Р и актуальными редакциями IEC — требует проверки.

## Ключевые понятия: Zone и Conduit

**Security Zone (зона безопасности)** — логическая группа активов с **одинаковыми требованиями к безопасности**. Все устройства в зоне имеют одинаковый уровень доверия.

Пример: ПЛК 1, ПЛК 2, ET 200SP 1–5 в одном цехе → одна зона «Управление участком А». SCADA-серверы → отдельная зона «Диспетчерское управление».

**Conduit (канал)** — **контролируемый путь** между двумя зонами для передачи данных. Канал определяет: что можно передавать, по каким протоколам, с какими ограничениями.

Если между зонами нет явного канала — трафик запрещён по умолчанию.

> [!ПРИМЕР]
> SCADA-сервер (зона L2) хочет читать данные из ПЛК (зона L1).
>
> Без IEC 62443: SCADA просто подключается к ПЛК по Modbus TCP — прямое соединение.
>
> По IEC 62443: определяется **канал** (conduit) между зоной L2 и зоной L1:
> - Протокол: только Modbus TCP, порт 502
> - Направление: только L2 → L1 (инициирует SCADA)
> - Ограничение: только чтение (FC03/04), запись запрещена
> - Контроль: межсетевой экран с правилами по conduit
>
> Всё остальное между зонами — запрещено по умолчанию.

## Security Levels (SL): уровни защиты

IEC 62443-3-3 вводит **Security Levels** — 4 уровня защиты, определяющих, против каких угроз защищена зона:

| SL | Уровень | Защищает от |
|----|---------|-------------|
| **SL 0** | Нет требований | — |
| **SL 1** | Базовый | Случайных/непреднамеренных воздействий, хакеров-любителей без мотивации |
| **SL 2** | Средний | Целенаправленных атак с умеренными ресурсами (обычные хакерские группы) |
| **SL 3** | Высокий | Атак с серьёзными ресурсами и экспертизой (APT-группы, Nation State) |
| **SL 4** | Критический | Атак государственного уровня с неограниченными ресурсами |

В промышленности обычно: SL 1 для периферийных систем, SL 2 для большинства OT, SL 3 для критической инфраструктуры.

## Как применить Zone/Conduit на практике

Алгоритм зонирования по IEC 62443-3-2:

1. **Инвентаризация активов** — список всех устройств, ПО, коммуникаций.
2. **Оценка угроз** (Threat Analysis): для каждого актива — что может произойти при компрометации.
3. **Группировка в зоны** по уровню доверия и функциональности. Правило: устройства с одинаковым владельцем, одинаковым уровнем защиты, в одном физическом сегменте — в одну зону.
4. **Определение каналов** между зонами: что, куда, по какому протоколу, в каком направлении.
5. **Назначение SL** каждой зоне исходя из оценки рисков.
6. **Реализация** — VLAN, межсетевые экраны, ACL, мониторинг согласно требованиям SL.

> [!ВАЖНО]
> IEC 62443 — это не обязательный регуляторный документ в России (за исключением случаев, когда его требует контракт или ссылается ГОСТ). Но это лучшая международная практика. На объектах КИИ требования ФСТЭК (Приказ №239) используют схожий подход зонирования — понимание IEC 62443 помогает выполнить российские требования осмысленно.
