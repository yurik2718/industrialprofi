# IndustrialProfi

**The Odin Project + roadmap.sh — для рабочих профессий.**

Бесплатная open-source платформа, которая учит индустриальным профессиям так,
как мастера учатся в реальности: читая официальные стандарты
(ГОСТ, ASME, НАКС) и выполняя проверяемые практические задания.

## Зачем это нужно

The Odin Project доказал: освоить full-stack веб-разработку можно бесплатно —
не через видеокурсы, а читая документацию и собирая реальные проекты.
**IndustrialProfi переносит эту модель в рабочие профессии.**

У сварщиков, электриков, специалистов КИПиА и других рабочих в СНГ
(Россия, Казахстан) нет структурированного, бесплатного пути развития на основе
стандартов. Официальные знания уже существуют — они в ГОСТ, ПУЭ, НАКС и ASME —
но разрознены, пугают объёмом и никем не разложены по полочкам.
IndustrialProfi собирает их в понятные карьерные роадмапы:

```
Профессия   →  Курс              →  Урок
(Электрик)     (Охрана труда)       (ПУЭ: Правила устройства электроустановок)
```

## Как устроен урок

Каждый урок следует одной честной структуре без воды — та же идея, что и в
задании Odin Project, но адаптированная под цех и площадку:

1. **ЗАЧЕМ** — одно-два предложения, почему это важно на работе.
2. **ОФИЦИАЛЬНЫЕ ДОКУМЕНТЫ** — выверенные ссылки на сами стандарты, с рангом
   (★ обязательно, ○ опционально). Без пересказа — ты читаешь первоисточник.
3. **ПРАКТИЧЕСКОЕ ЗАДАНИЕ** — конкретное, проверяемое.
4. **✓ Отметить выполненным** — бинарный прогресс, как в Odin. Сделано или нет.

Прогресс по курсу — это просто `выполнено / всего`. Никаких «в процессе», никакой
ложной геймификации — только ясная карта того, что ты знаешь и что дальше.

## Стек

Это намеренно «скучное» Rails-приложение **без шага сборки** — и в этом смысл.
Оно показывает, как много можно сделать на чистом Rails 8 и Hotwire, не имея
Node.js нигде.

- **Ruby 4.0 / Rails 8.1**
- **SQLite3** + Solid Queue, Solid Cache, Solid Cable
- **Hotwire** (Turbo + Stimulus) — серверный рендеринг HTML, без SPA
- **Чистый CSS**, отдаётся как есть через **Propshaft** — без Tailwind, PostCSS,
  бандлера и сборки. Каскад — это просто имена файлов в алфавитном порядке.
- **Importmap** — без Node, Webpack и Vite
- Авторизация через `has_secure_password` (bcrypt) — без Devise
- **Kamal 2** + Docker + Thruster для деплоя
- **Minitest** + фикстуры + Capybara
- Только системные шрифты, цветовые токены OKLCH, светлая тема по умолчанию с
  автоматической тёмной — паттерны UI повторяют open-source приложения Basecamp.

## Быстрый старт

Нужны Ruby 4.0.5 и Git. Никакого Node, Yarn или пайплайна ассетов настраивать
не нужно.

```bash
git clone https://github.com/yurik2718/industrialprofi.git
cd industrialprofi
bin/setup          # ставит гемы, готовит базу, заливает примеры данных
bin/dev            # запускает сервер на http://localhost:3000
```

Частые команды:

```bash
bin/rails test          # прогон тестов
bin/rails test:system   # системные тесты (Capybara)
bin/rails db:migrate    # миграции
bin/rubocop             # линтер
```

## Структура проекта

```
app/models/                 # доменные модели (Path, Course, Lesson, ...)
app/controllers/            # RESTful-контроллеры, рендерят ERB
app/views/                  # ERB-шаблоны + партиалы Turbo Frame/Stream
app/javascript/controllers/ # Stimulus-контроллеры
app/assets/stylesheets/     # весь CSS — по одному самодостаточному файлу на компонент
db/migrate/                 # миграции = источник истины для схемы
docs/                       # VISION.md, MVP.md, заметки по дизайну
```

Иерархия контента:

```
Path (профессия)  →  Course  →  Lesson  →  Resource (ссылки на стандарты)
User  →  LessonCompletion  (бинарно: строка существует = урок пройден)
```

- **Контент** — программа обучения и уроки под

Простыми словами: ты свободен использовать, изучать, распространять и изменять
**код** — а если запускаешь изменённую версию как сетевой сервис, обязан тоже
открыть свой исходный код. **Учебный контент** ты можешь свободно распространять

IndustrialProfi и сохранения той же лицензии. Сторонние стандарты (ГОСТ, ASME,
НАКС…) мы только цитируем и на них ссылаемся — они принадлежат своим издателям.
